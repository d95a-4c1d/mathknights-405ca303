# Mathknights 项目方案书
**—— 基于大模型与二游养成机制的高数辅助学习智能体**

## 一、 应用场景需求分析

### 1.1 大学新生的“学习迷茫期”
当代大学新生刚刚从高中的集中化的、目标明确的教育管理体系中脱离，进入到以自主探索、自我规范为主的大学以后，在学习与生活往往陷入“不知道要做什么”、“不知道在忙什么”、“不知道要怎么做”的“三不知”迷茫境地。而在生活中失去方向往往让人沉溺于即时的娱乐活动——短视频、网络游戏、小说动漫、电视剧等，逐渐颓废，形成了恶行循环。

这样大把的青春时光被浪费掉，着实令人唏嘘。为了重新激发同学们的学习热情，改变现有不利局面，我们就需要建立一套行之有效的即时反馈与目标明确机制。而这一套机制要如何建立呢？答案其实就在电子游戏本身之中。在我们一般的印象里，电子游戏与学习科研似乎是互斥的，但在我看来，它们有着一些相似之处。

### 1.2 游戏和学习是契合的
首先，游戏和学习都有很强的交互性。区别于其它的商业文化产品，游戏不仅结合了音乐、美术、文学等多种艺术形式，还具有交互性，玩家可以参与到游戏叙事之中，改变剧情走向，增强了玩家的体验感、沉浸感。而学习的过程也是一个交互的过程。在解题时，我们需要发挥主观能动性来生成答案；在请教别人或者 AI 时，我们需要思考提问方式，分析他们给出的回答，并修正自己的知识体系或继续向他们追问；在课余时间我们还需要主动规划学习，制定学习计划、养成良好学习习惯。

其次，游戏和学习还都有很强的时间占有。实际上，占有玩家的空余时间，就是电子游戏的设计目标之一。玩家在某一款游戏里投入的时间越长，就越有可能在这款游戏中消费、创造社区讨论度和产生流量，而不是在其它游戏中。统计数据表明，通关一个线性叙事的游戏往往要花费20~70个小时，而在亚洲地区盛行的内购制运营模式的游戏则更甚——根本没有“通关”的概念，通过日常任务、限时活动和签到等手段吞噬了玩家每周几十个小时的时间。而学习在时间占用上也是巨大的。在现有的教育体制下，学生需要花费至少9年的时间完成基础的义务教育，大多数人还会继续上高中、大学，再投入7~13年的时间。而这些教育都是通过全日制学校完成的，再加上补习班、补课等，学习占用了学生的大部分生活。

### 1.3 一个全新的学习范式

这样的相似性启发了我的思考，让我得以提出一个新的学习范式——二游养成式学习。这一范式的提出基于两点预设：第一，学习过程和电子游戏相似，应该被鼓励为交互式的、创新性的、趣味性的、沉浸式的；第二，学习过程需要有效的激励机制，以支持学生投入大量的时间。而我结合自身游玩的阅历和目前网络游戏的市场情况，发现公认最为暴利的二次元题材游戏中的角色养成系统刚好是十分适用于激励我们的学习的。

一个角色养成系统一般包含以下几个要素：有强大魅力的游戏角色本身；一个任务系统用以针对性地展示游戏内容；一个日常活动系统以在游戏开发团队产出较少的“长草期”继续收割玩家时间创造日活；一个核心玩法来提供电子游戏特有的交互性、一个角色养成界面以变现玩家的投入，为角色添加新的能力、新的剧情与立绘，增加角色价值。这些要素共同构成了一个以角色为核心的注意力收割流水线，其激励玩家投入时间的效果不言而喻。

因此，我们同样有机会利用这个系统，将注意力重新转移到学习上。

## 二、 技术路线

### 2.1 整体设计

由于专长不在游戏开发方面且时间有限，我决定先从单独的高数学习入手，构建一个高等数学学习计划辅助应用。我设计了一个与之类似的系统。该系统由日常活动、任务手册、趣味玩法和能力评估几个功能组成，分别对应二游的日常活动、任务系统、核心玩法和角色养成。

具体地，日常活动功能对学生课余时间作出了日常性的规划，包括每日做题、公式默写、错题回顾等日常学习活动；任务手册则为学生提供了时间规划参考与明确的目标，鼓励学生有计划、持之以恒地学习；趣味玩法则类似游戏关卡，学生可以按部就班地根据课程进度来学习来获得奖励、完成日常活动；能力评估将以高等数学的几个核心学科素养的掌握程度为指标，展示学生本人的整体高数水平，激励学生的学习行为。

本项目采用**前后端分离 + 大模型云端服务**的现代化架构，确保系统的高响应速度、交互的丰富性以及未来扩展潜力。

### 2.2 技术架构
* **前端**：采用 React + Vite 框架，结合 Tailwind CSS + DaisyUI 组件库
* **后端**：基于 FastAPI (Python) 搭建，负责用户认证、数据库管理、以及大模型记忆流。
* **LLM API**：接入 DeepSeek V3.2 API，利用其数理逻辑较强、性价比高的特点，承担题目生成和答案检查工作。

### 2.3 大模型核心集成点
- **题目生成**：题目生成是应用的核心能力之一，用于生成主线关卡每次挑战的题目。
- **答案检查**：学生每次过关时可以检查学生答案是否正确，还可以提出错误所在。

## 三、 创新点说明

1. 结合了二游养成体系，利用了它的注意力收割能力和规划能力来产生正面激励效果。
2. 大量使用 AIGC 产出内容，既避免了撞题等情况，又使得题目多变，不枯燥，还减少了人工维护题库的成本。

## 四、 使用示例
1. **主线通关功能**
   学生进入“第一章：函数与极限”地图。系统调用大模型生成一组当前难度的挑战题。学生沉浸式解题并提交答案，大模型实时批改。通关后，界面弹出“战斗胜利”动效，发放经验值奖励，并解锁下一知识点节点。
2. **任务领取功能**
   学生在课余时间打开应用，完成任务后获得任务积分，进而获得养成奖励，填补无课时的“长草期”，培养持之以恒的学习习惯。
3. **自选题目功能**
   期中考试前，学生发现自己“多重积分”较为薄弱。在自选模块中设定“多重积分、中等难度、5道题”，系统即刻生成专属特训副本供学生定向强化。
4. **能力养成功能**
   学生在主页查看自己的“高数能力雷达图”（涵盖计算、逻辑推导、空间想象等维度）。随着主线通关和日常打卡，雷达图各项数值肉眼可见地增长，将抽象的“知识积累”具象化为面板属性的提升。
