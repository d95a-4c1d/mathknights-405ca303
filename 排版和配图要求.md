# Mathknights 项目PPT大纲
**—— 基于大模型与二游养成机制的高数辅助学习智能体**

> **【封面设计】** > * **页面设计**：居中大标题，副标题紧随其后。整体采用“赛博朋克”或“二次元科幻”与“学术”结合的风格，背景色建议采用深色调（如深蓝、暗紫）以突出发光元素。
> * **配图要求**：设计一个“Mathknights”的Logo（带着学士帽的二次元美少女骑士）。背景可以隐约透出复杂的数学公式和游戏UI的代码流。

## 一、 应用场景需求分析

### 1.1 大学新生的“学习迷茫期”

#### 第1页
当下大学新生的困境：“不知道要做什么”、“不知道在忙什么”、“不知道要怎么做”。而在生活中失去方向往往让人沉溺于即时的娱乐活动——短视频、网络游戏、小说动漫、电视剧等，逐渐颓废，形成了恶行循环。
这样大把的青春时光被浪费掉，着实令人唏嘘。我们就需要建立一套行之有效的即时反馈与目标明确机制。
> * **【页面设计】**：左右分栏排版。左侧为文字痛点描述，字号递进放大，强调“恶性循环”；右侧放置核心配图。
> * **【配图要求】**：一张极具表现力的概念图或拼图：中心是一个面露迷茫、抱头坐在书桌前的大学生，周围被各种发光的图标（抖音、游戏手柄、动漫角色）形成一个“漩涡”或“锁链”包围。

#### 第2页
而这一套机制要如何建立呢？答案其实就在电子游戏本身之中。在我们一般的印象里，电子游戏与学习科研似乎是互斥的，但在我看来，它们有着一些相似之处。
> * **【页面设计】**：居中强调式排版。利用动画效果，先抛出问题，再引出“电子游戏”这个反直觉的答案。
> * **【配图要求】**：一个打破刻板印象的创意图形。左边是传统的厚重教科书，右边是炫酷的游戏手柄，中间用一个带有问号的桥梁或发光的“VS”转变成“=”的动效图连接。

### 1.2 游戏和学习是契合的

#### 第3页
首先，游戏和学习都有很强的交互性。
区别于其它的商业文化产品，游戏不仅结合了音乐、美术、文学等多种艺术形式，还具有交互性，玩家可以参与到游戏叙事之中，改变剧情走向，增强了玩家的体验感、沉浸感。
> * **【页面设计】**：卡片式排版。提炼出“音乐”、“美术”、“文学”、“交互”四个关键字做成并列的图标卡片。
> * **【配图要求】**：一张高质量的二游截图（如《原神》或《崩坏：星穹铁道》的剧情对话界面），重点用高亮红框圈出“玩家的选择分支”，体现交互性。

#### 第4页
而学习的过程也是一个交互的过程。在解题时，我们需要发挥主观能动性来生成答案；在请教别人或者 AI 时，我们需要思考提问方式，分析他们给出的回答，并修正自己的知识体系或继续向他们追问；在课余时间我们还需要主动规划学习，制定学习计划、养成良好学习习惯。
> * **【页面设计】**：环形闭环流程图。突出学习是一个“输入-思考-输出-反馈”的循环。
> * **【配图要求】**：一个学生正在与悬浮的AI大模型（DeepSeek形象）对话的插画，旁边标注出“提问 -> 分析 -> 修正 -> 追问”的飞线动效。

#### 第5页
其次，游戏和学习还都有很强的时间占有。统计数据表明，通关一个线性叙事的游戏往往要花费20~70个小时，而在亚洲地区盛行的内购制运营模式的游戏则更甚——根本没有“通关”的概念，通过日常任务、限时活动和签到等手段吞噬了玩家每周几十个小时的时间。
> * **【页面设计】**：数据图表主导的排版。用强烈的对比色突出二游极强的时间占用率。
> * **【配图要求】**：柱状图或面积图。对比“传统单机游戏（20-70h）”与“二游（无穷大，通过体力系统/日常延长寿命）”。可配上经典二游的“体力值/理智值不足”的UI截图作为点缀。

#### 第6页
而学习在时间占用上也是巨大的。在现有的教育体制下，学生需要花费至少9年的时间完成基础的义务教育，大多数人还会继续上高中、大学，再投入7~13年的时间。而这些教育都是通过全日制学校完成的，再加上补习班、补课等，学习占用了学生的大部分生活。
> * **【页面设计】**：时间轴（Timeline）排版。将学习生涯游戏化。
> * **【配图要求】**：一条类似游戏“经验条”或“升级路线图”的时间轴：Lv.1-9（义务教育） -> Lv.10-12（高中） -> Lv.13-16（大学），并在上方漂浮各种副本图标（如：期末考副本、考研深渊）。

### 1.3 一个全新的学习范式

#### 第7页
这样的相似性启发了我的思考，让我得以提出一个新的学习范式——二游养成式学习。
这一范式的提出基于两点预设：第一，学习过程和电子游戏相似，应该被鼓励为交互式的、创新性的、趣味性的、沉浸式的；第二，学习过程需要有效的激励机制，以支持学生投入大量的时间。
而我发现二次元题材游戏(以下简称二游)中的角色养成系统刚好是十分适用于激励我们的学习的。
> * **【页面设计】**：全屏大字报+两根支柱的结构。强调“二游养成式学习”这个核心概念。
> * **【配图要求】**：一张视觉冲击力强的概念合成图：一个动漫角色正在“升级进阶”（周围环绕金光和UP字样），但仔细看，角色身上掉落或吸收的经验球全部都是数学公式（极限、导数、积分）。

#### 第8页
一个角色养成系统一般包含以下几个要素：游戏角色本身；一个任务系统；一个日常活动系统；一个核心玩法；一个角色养成界面。这些要素共同构成了一个以角色为核心的注意力收割流水线。
因此，我们同样有机会利用这个系统，将注意力重新转移到学习上。
> * **【页面设计】**：雷达图或中心散射图排版。中心是“角色（学生）”，四周散射出五大要素。
> * **【配图要求】**：绘制一个“注意力收割流水线”的齿轮模型或能量汇聚图。游戏角色/玩家在中央，周围是代表“日常”、“任务”、“核心玩法（战斗/解题）”、“养成面板”的模块，共同向中心输送“注意力/经验值”。

## 二、 技术路线

### 2.1 整体设计

#### 第9页
我决定先从单独的高数学习入手，构建一个高等数学学习计划辅助应用。
我设计了一个与之类似的系统。该系统由日常活动、趣味玩法和能力评估几个功能组成，分别对应二游的日常活动、核心玩法和角色养成。
> * **【页面设计】**：对比映射图（Mapping）排版。展示二游与高数App的一一对应关系。
> * **【配图要求】**：连线对比图。
>   左侧（二游要素）：清日常、打主线关卡、角色面板突破。
>   右侧（Mathknights）：公式默写/每日5题、高数章节闯关、高数能力雷达图。中间用发光的箭头连接。

#### 第10页
具体地，日常活动功能对学生课余时间作出了日常性的规划，包括每日做题、公式默写、错题回顾等日常学习活动；
趣味玩法则类似游戏关卡，学生可以按部就班地根据课程进度来学习来获得奖励、完成日常活动；能力评估将以高等数学的几个核心学科素养的掌握程度为指标，展示学生本人的整体高数水平。
> * **【页面设计】**：三列式陈列。每一列对应一个核心功能点。
> * **【配图要求】**：放置三个精美的UI草图/原型图。
>   图1：带有签到日历和任务列表的手机UI。
>   图2：类似《明日方舟》主线地图的UI（节点是“洛必达法则”、“泰勒公式”）。
>   图3：一个六边形能力雷达图。

### 2.2 技术架构

#### 第11页
本项目采用**前后端分离 + 大模型云端服务**的架构。
* **前端**：采用 React + Vite 框架，结合 Tailwind CSS + DaisyUI 组件库
* **后端**：基于 FastAPI (Python) 搭建，负责用户认证、数据库管理、以及大模型记忆流。
* **LLM API**：接入 DeepSeek V3.2 API，利用其数理逻辑较强、性价比高的特点，承担题目生成和答案检查工作。
> * **【页面设计】**：经典的技术架构图（Architecture Diagram）布局，分层清晰（前端、后端、AI层）。
> * **【配图要求】**：使用各大技术的官方Logo（React, Vite, Tailwind, FastAPI, Python, DeepSeek）绘制一张现代风格的技术架构图。突出DeepSeek大模型在架构中的“核心大脑”地位。

### 2.3 大模型核心集成点 & 2.4 做题时的时序图

#### 第12页
- **题目生成**：用于生成主线关卡每次挑战的题目。
- **答案检查**：学生每次过关时检查答案是否正确，并指出错误所在。
**做题时的时序图：**
- 请求题目数据: 前端 -> 后端
- 展示题目信息: 后端 -> 前端
- 生成题目: 后端 -> LLM API
- 展示题目: 后端 -> 前端
- 学生完成以后提交: 前端 -> 后端
- 判题: 后端 -> LLM API
- 展示题目完成情况和奖励: 后端 -> 前端
> * **【页面设计】**：大图占主体，文字做旁白解释。突出程序的严密性和逻辑性。
> * **【配图要求】**：一张标准的UML时序图（Sequence Diagram）。包含三个参与者（泳道）：前端APP、后端服务器、DeepSeek大模型。用显眼的颜色标出“生成题目”和“判题反馈”这两个调用LLM的关键动作。

## 三、 创新点说明

#### 第13页
1. 结合了二游养成体系，利用了它的注意力收割能力和规划能力来产生正面激励效果。
2. 大量使用 AIGC 产出内容，既避免了撞题等情况，又使得题目多变，不枯燥，还减少了人工维护题库的成本。
> * **【页面设计】**：大字号双核心卖点排版。页面一分为二，左边谈“机制创新”，右边谈“技术创新”。
> * **【配图要求】**：
>   左侧图：一个胡萝卜挂在机械臂上（隐喻正反馈激励）或一个游戏连抽获得SSR金光的特效。
>   右侧图：一个机器人正在飞速打印出无尽且不重复的数学卷子（隐喻AIGC无限题库）。

## 四、 使用示例

> **【本章整体建议】**：第四章是产品展示环节，页面背景建议换成深色手机壳或平板电脑边框，所有配图都应以“应用界面原型图（Mockup）”的形式嵌入在设备框内，增强真实感。

#### 第14页：主线通关功能
学生进入“第一章：函数与极限”地图。系统调用大模型生成一组当前难度的挑战题。学生沉浸式解题并提交答案，大模型实时批改。通关后，界面弹出“战斗胜利”动效，发放经验值奖励，并解锁下一知识点节点。
> * **【页面设计】**：左侧文字说明，右侧放手机横屏模式的产品界面。
> * **【配图要求】**：游戏战斗结算界面的Mockup。画面上方是巨大的“VICTORY（战斗胜利）”字样，中间是解开的极限题目，下方是掉落的战利品（如：金币+50，积分学识+100），背景是解锁进度条填满的特效。

#### 第15页：任务领取功能
学生在课余时间打开应用，完成任务后获得任务积分，进而获得养成奖励，填补无课时的“长草期”，培养持之以恒的学习习惯。
> * **【页面设计】**：竖屏手机应用展示风格。
> * **【配图要求】**：类似二游日常任务界面的Mockup。列表项包括：“登录游戏（10活跃度）”、“手刃三道微积分（30活跃度）”、“温故知新：复习错题本（20活跃度）”。上方有100活跃度解锁的“钻石宝箱”。

#### 第16页：自选题目功能
期中考试前，学生发现自己“多重积分”较为薄弱。在自选模块中设定“多重积分、中等难度、5道题”，系统即刻生成专属特训副本供学生定向强化。
> * **【页面设计】**：突出自定义配置的交互感，配图可带箭头指向生成的副本。
> * **【配图要求】**：“生成副本”界面的Mockup。界面中有精美的下拉菜单（选择知识点：多重积分）、滑动条（难度：Lv.50 中等）、数量选择（5道）。底部是一个巨大的、发着魔法光芒的“生成特训副本（消耗体力20）”按钮。

#### 第17页：能力养成功能
学生在主页查看自己的“高数能力雷达图”（涵盖计算、逻辑推导、空间想象等维度）。随着主线通关和日常打卡，雷达图各项数值肉眼可见地增长，将抽象的“知识积累”具象化为面板属性的提升。
> * **【页面设计】**：大图居中，突出成就感和养成感。这是整个展示的高潮，也是“正反馈”的直观体现。
> * **【配图要求】**：角色个人属性面板的Mockup。左侧是用户自定义的二次元数字分身（或默认的炫酷头像），右侧是一个动态的六维雷达图（计算、逻辑推导、空间想象、概念理解、应用能力、错题免疫）。可以做一个“升级前（小面积）”叠加“升级后（大面积并发光）”的对比图，体现出明显的成长感。
